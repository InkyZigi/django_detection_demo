System Design

Algorithm Design: Several methods of algorithms has been listed as followed.

CNN

Convolutional Neural Network (CNN) is a deep learning model that can extract and learn image features through multi-layer neural network structures such as convolutional layers and pooling layers, thereby completing tasks such as image classification or semantic segmentation.

VGG-19

The development of convolutional neural networks has gone through many stages, and the VGG model is an important milestone model. VGG19 is a representative model of the VGG series, with a simple and clear structure. It is composed of 16 stacked convolutional layers with the same kernel, and has achieved good generalization performance on the ImageNet dataset.

Residual Network

Residual Network is a deep neural network architecture designed primarily to address gradient vanishing or exploding problems in deep neural networks, as well as mitigate potential model degradation issues.

On the basis of the VGG19 network model, the residual network adds a short circuit mechanism between every two or three layers of the ordinary network (Plain Networks), and through residual deep learning, the model performs better.

Knowledge Distillation

Knowledge distillation is a type of teacher student training network structure that involves distillation training on a large and complex model (teacher model) that has been trained, transferring knowledge to a small, structurally simple model (student model), and obtaining a smaller scale, less computational resource required, and still maintaining original performance at the cost of slight performance loss.

The defect detection based on the student teacher network model mainly utilizes pixel level discriminative embedding features extracted from the pre trained network (teacher model) as input data, and performs distillation training on another relatively simple model (student model). The training of a model is actually to transform the problem of image defect detection into a problem of feature regression. On training data without anomalies, the student model simulates the output of the teacher model, so that the student model only provides erroneous predictions on defect data, thereby achieving recognition of abnormal data.

System Design: How our system are constructed?

System Architecture Design: Website and Database.

The website is based on Django's MVT architecture, which encapsulates the page information of the embedded template engine through templates; Views are responsible for business processing, receiving HTTP requests and responses; The model is based on the ORM framework to achieve database interaction.

After receiving requests from the client layer, the system transmits the data to the backend with the support of the template engine. It then provides reasonable feedback based on the different business needs of users, and finally calls the database interface on the data layer to process the data.

System Module Design: Provides Users with Interaction-rich Modules

The production management module allows users to operate the interactive components of web pages to manage work tasks and form a production dashboard for data visualization.

The model management module provides users with various algorithmic interactions and real-time monitoring functions, and additionally provides API data interfaces.

The data retrieval module, based on database management, can provide users with functions for data search and statistics. Report management provides users with task summary functionality.

The report management module provides functions for data analysis and summary.

System Database Design

The overall E-R diagram of the system includes multiple entities such as employees, departments, enterprises, work orders, models, and approval forms, as well as various attributes within each entity and their relationships with different entities.
